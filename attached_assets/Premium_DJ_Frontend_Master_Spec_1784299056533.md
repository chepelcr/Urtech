# Premium Electronic DJ Website --- Frontend Specification for Replit Agent

> **Goal:** Build a production-quality **frontend-only** prototype. Do
> **not** implement a backend, database, authentication, or APIs. Use
> mock data and realistic placeholders. The result should look ready to
> launch and suitable for client presentation.

# Team

-   Senior Product Designer
-   Senior UI/UX Designer
-   Art Director
-   Frontend Architect
-   React/Next.js Engineer
-   Motion Designer
-   Accessibility Specialist
-   SEO Specialist

# Creative Direction

Create an immersive, premium experience for an underground electronic
music artist.

Keywords: - Minimal - Black & White - Elegant - Industrial - Cinematic -
Futuristic - Underground - Premium

Avoid colorful gradients, clutter and generic templates.

# Technology

-   Next.js
-   React
-   TypeScript
-   Tailwind CSS
-   Framer Motion

Frontend only. Mock all data.

# Design System

Create reusable components and design tokens: - Typography scale - 8px
spacing system - Black/white/gray palette - Buttons - Cards - Inputs -
Badges - Tags - Dividers - Modals - Drawers - Tooltips - Toasts -
Skeleton loaders - Empty states - Error states

# Components

Build reusable: - Navbar - Hero - Music Cards - Mix Cards - Video

# Motion

Use Framer Motion: - Fade - Slide - Scale - Stagger - Smooth page
transitions - Hover micro-interactions - Smooth scrolling - Elegant menu
animations

No excessive animation.

# Audio Experience

Persistent bottom player with: - Play/Pause - Progress - Volume - Queue
UI - Waveform visualization - Track artwork - Continues across page
navigation (frontend simulation)

# Music Platform Integrations (UI Prototype)

Create UI that simulates: - SoundCloud profile and latest mixes -
Spotify releases and playlist - YouTube live sets - Instagram gallery -
Beatport releases - Resident Advisor profile

Use realistic mock data and architecture prepared for future API
integration.

# SEO (Frontend)

Prepare: - Meta titles - Meta descriptions - Open Graph placeholders -

# Code Quality

Feature-based folder structure. Reusable components. Strict TypeScript.
Clean architecture. No duplicated code.

# Acceptance Criteria

The prototype should: - Feel like an internationally recognized DJ
brand. - Be visually polished. - Be fully responsive. - Include polished
loading, empty and hover states. - Use only mock data. - Be
presentation-ready with no backend dependencies.

Deliver a polished, production-style frontend experience.
